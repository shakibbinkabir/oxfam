## Data Entry — Multi-Step Wizard Form

### 1. Wizard Form Backend Support

- [ ] `POST /api/v1/risk-index/` — Create a full Risk Index record (all 40+ indicators for one union at one time)
  - **Request body:**
    ```json
    {
      "boundary_pcode": "BD30140602",
      "year": 2024,
      "values": {
        "rainfall": 3.5,
        "heat": 38.2,
        "literacy": 52.0,
        ...
      }
    }
    ```
  - Validates every indicator code against `climate_indicators` table
  - Range-checks each value against `indicator_reference.global_min / global_max` (warn if outside range, don't reject)
  - Inserts/upserts all values in a single transaction
  - Returns computed CVI scores for the submitted data
- [ ] `PUT /api/v1/risk-index/{boundary_pcode}` — Update all indicator values for a boundary (admin only)
  - Creates audit version before overwrite (see Audit Trail below)

### 2. Multi-Step Wizard Frontend

- [ ] New route: `/dashboard/submit-risk-index` (replace current single-indicator submit page)
- [ ] **Step 1 — Location Selection:**
  - Cascading dropdowns: Division → District → Upazila → Union
  - Year selector (default: current year)
  - If union already has data: show warning "Data exists for this union. Submitting will update values."
  - "Next" button
- [ ] **Step 2 — Hazard Indicators (9 fields):**
  - rainfall, heat, colddays, drought, water, erosion, surge, salinity, lightning
  - Each field: label, unit hint, source reference, numeric input
  - Validation: required, numeric, within plausible range (show warning icon if outside global min/max)
  - Progress bar showing "Step 2 of 5"
- [ ] **Step 3 — Socioeconomic Data:**
  - Exposure (4 fields): population, household, female, child_old
  - Sensitivity (13 fields): pop_density, dependency, disable, unemployed, fm_ratio, vulnerable_hh, hh_size, slum_float, poverty, crop_damage, occupation, edu_hamper, migration
  - Same field format as Step 2
- [ ] **Step 4 — Adaptive Capacity & Environmental Data:**
  - Adaptive Capacity (17 fields): literacy through pavedroad
  - Environmental Exposure (3 fields): forest, waterbody, agri_land
  - Environmental Sensitivity (3 fields): ndvi, wetland_loss, groundwater
- [ ] **Step 5 — Review & Submit:**
  - Summary table of all entered values grouped by dimension
  - **CVI Preview:** Call simulation endpoint (not persisting) to show computed scores before save
  - Scores displayed: Hazard, Exposure, Sensitivity, AC, Vulnerability, CRI
  - Category label and colour for CRI
  - "Confirm & Submit" button → calls `POST /api/v1/risk-index/`
  - "Back" button to return and edit values
  - Success confirmation modal with option to "Enter Another" or "View on Map"

### 3. Edit Existing Record

- [ ] From indicator values list, admin can click "Edit" on a boundary's dataset
- [ ] Opens wizard pre-populated with existing values
- [ ] On submit: calls `PUT /api/v1/risk-index/{pcode}` (creates audit version)

## Export Functionality

### 4. CSV Export — Risk Index Data

- [ ] `GET /api/v1/export/csv?level={level}&division_pcode={pcode}` — Export risk index data
  - Columns: boundary_pcode, boundary_name, division, district, upazila, all 40+ indicator values, all computed scores (hazard, exposure, sensitivity, ac, vulnerability, cri)
  - Filter by admin level and/or parent boundary
  - Streaming response for large datasets
- [ ] Frontend: "Export CSV" button on map dashboard and indicator values page
- [ ] Download triggers browser file save dialog

### 5. PDF Export — Area Report

- [ ] `GET /api/v1/export/pdf?boundary_pcode={pcode}` — Generate area-level PDF report
- [ ] **Report contents (single page per boundary):**
  - Header: CRVAP logo + area name + location breadcrumb
  - CRI score prominently displayed with category label
  - Dimension scores as bar chart (use matplotlib or ReportLab graphics)
  - Table of all 40+ raw indicator values with normalised scores
  - Calculation methodology note (brief)
  - Footer: "Generated by CRVAP on {date}" + data timestamp
- [ ] Backend: Use **ReportLab** library for PDF generation
  - Add `reportlab` to pyproject.toml dependencies
- [ ] Frontend: "Download PDF" button in Detail Side Panel
- [ ] Loading spinner while PDF generates

### 6. Shapefile Export (Admin Only)

- [ ] `GET /api/v1/export/shapefile?level={level}&indicator={indicator}` — Export as Shapefile
- [ ] **Output:** ZIP file containing .shp, .shx, .dbf, .prj files
  - Geometry: boundaries at specified admin level
  - Attributes: pcode, name, all component scores, CRI
- [ ] Backend: Use **Fiona + Shapely** (already installed) for Shapefile generation
  - Add `fiona` to pyproject.toml if not present
  - Write to temp directory, ZIP, stream response, cleanup
- [ ] Frontend: "Export Shapefile" button on map (admin only, hidden for other roles)
- [ ] File size warning for union-level export (potentially large)

- [ ] Add `is_deleted` (boolean, default false) and `deleted_at` (timestamp, nullable) columns to `indicator_values` table via migration

## Audit Trail

### 9. Audit Log Table

- [ ] Create `audit_logs` table via Alembic migration:
  ```
  id (UUID), user_id (FK users), action (enum: create, update, delete, restore),
  entity_type (string: indicator_value, indicator, user, scenario),
  entity_id (string), old_values (JSONB, nullable), new_values (JSONB, nullable),
  ip_address (string), user_agent (string), timestamp (datetime)
  ```
- [ ] Index on: `user_id`, `entity_type`, `timestamp`

### 10. Audit Logging Middleware

- [ ] Create `app/middleware/audit.py` or decorator-based approach
- [ ] Automatically log all write operations (POST, PUT, DELETE) on:
  - Indicator values (create, update, delete, restore)
  - Indicators (create, update, delete)
  - Users (create, update, deactivate)
  - Scenarios (create, delete)
- [ ] Capture: requesting user, IP address, before/after values (JSON diff)
- [ ] Log entry created within the same database transaction as the operation

### 11. Audit Log API & UI

- [ ] `GET /api/v1/audit-logs` — List audit logs (admin only)
  - Filters: user_id, entity_type, action, date range
  - Pagination
  - Returns: user name, action, entity description, timestamp, changed values
- [ ] Frontend: New route `/dashboard/audit-log` (admin only)
- [ ] Sidebar menu item: "Audit Log" under admin section
- [ ] Table view with:
  - Columns: Timestamp, User, Action, Entity, Changes
  - Changes column: expandable JSON diff (old → new)
  - Search and date range filter
  - Export audit log as CSV

## Acceptance Criteria

1. Admin can enter all 40+ indicator values for a union via a 5-step wizard and see computed CVI before submitting.
2. "Download PDF" in the Detail Side Panel generates a formatted PDF report with CRI score, dimension breakdown, and raw indicator table.
3. "Export CSV" downloads a CSV with all indicator values and computed scores for visible boundaries.
4. Admin can export Shapefile with boundary geometries and CRI scores.
5. Deleting an indicator value soft-deletes it (is_deleted = true); admin can restore it.
6. All create/update/delete operations are recorded in the audit log with user, timestamp, and changed values.
7. Batch upload accepts both CSV and Excel files and validates ranges against indicator reference.
